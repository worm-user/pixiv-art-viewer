# Pixiv Reference Manager - Requirements & Specifications

このドキュメントは、以前の会話に基づくPixiv Reference Managerの要件と実装計画のメモです。

## 1. Architecture & Technology Stack
- **Framework**: Electron + React (Vite)
- **Styling**: Vanilla CSS / CSS Modules
  - **デザイン方針**: 画像自体の色合いや印象を邪魔しないよう、完全な無彩色（グレースケールベースのダークテーマ等）でUIを構築します。
- **Pixiv Integration**:
  - アプリケーション内にPixivログイン用の別ウィンドウ（BrowserWindow）を立ち上げ、ユーザーがログインすると自動的に内部でリフレッシュトークンを取得・保存するシームレスな仕組みを組み込みます。
- **AI Tagging**:
  - ローカルの `Ollama` を使用。
  - **モデル**: 検閲による拒否（False positive）を回避し、イラストのタグ付けに最適な **Qwen2-VL（Abliterated / Uncensored などのファインチューンモデル）** をターゲットとします。
- **Color Analysis**: `node-vibrant` 等を使用して画像の主要な色（16進数）を抽出。
- **保存先**: OS標準の `Pictures` (ピクチャ) フォルダ内の専用ディレクトリ（Windows環境を想定）。

## 2. 実装機能一覧

### Electronメインプロセス (Backend)
- **組み込みPixivログイン機能**: Electronの機能を使ってPixivのログインページを表示し、認証成功後にトラフィックやセッションからリフレッシュトークンを自動抽出して安全に保存するフローを実装。
- **画像ダウンロード機能**: 取得したトークンを用いてPixiv APIを叩き、Picturesフォルダに画像を保存。
- **Ollama連携**: 画像をQwen2-VLモデルに渡し、以下の2つの軸でタグを出力させるプロンプトを構築。
  1. **Danbooruスタイルタグ**: `1girl`, `blue hair`, `school uniform` などの要素タグ。
  2. **絵の練習用タグ**: 構図、ライティング、塗りのスタイル、パースなどの技術的な特徴タグ。

### フロントエンドUI (React)
- **タブベースの画像表示**: 開いている画像は上部のタブで切り替えられるようにします（Photoshopのようなイメージ）。
- **トグル式のファイルブラウザ**: フォルダ内の画像を一覧表示・検索できるサイドバー（またはオーバーレイウィンドウ）を実装し、不要な時は非表示（トグル）にできるようにします。
- **画像プレビューウィンドウ**:
  - メイン画像の表示（タブ切替対応）。
  - 右側または下部のパネルに、抽出した「Danbooruタグ」と「練習用スタイルタグ」を表示。
  - カラーパレット表示（無彩色のUIの中で抽出した色だけが目立つように配置）。
  - クリップボードへのコピーボタン。
  - スポイト機能（画像上の任意のピクセルをクリックして色を取得）。
